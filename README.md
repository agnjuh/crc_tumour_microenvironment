# CRC-TME: Morphological State Mapping of the Colorectal Tumour Microenvironment

## Overview

This project investigates whether quantitative morphology and nuclear architecture can resolve tissue states within the colorectal tumour microenvironment.

Using H&E-stained image patches from the CRC-VAL-HE-7K dataset, an integrated feature space was constructed from classical morphology descriptors and nucleus-derived measurements. This representation was used to classify colorectal tissue compartments, construct morphology atlases, identify tumour-stroma interface regions, and derive a continuous normal-to-tumour morphology progression axis.

The analysis demonstrates that colorectal tissue morphology occupies a structured state space in which normal and tumour-associated phenotypes form partially overlapping but distinguishable regions connected by intermediate morphological states.

## Feature extraction

Image-level morphology descriptors were extracted from each patch, including colour intensity, intensity variation, entropy, edge density and texture-related measurements.

Nuclear morphology was quantified using classical image processing and connected-component analysis. Extracted nuclear measurements included nuclei count, nuclei density, nuclear area fraction, mean nuclear area, nuclear size variability and nuclear shape descriptors.

The final integrated feature matrix contained morphology-level and nucleus-level measurements for all 7,180 patches.

| Feature matrix | Value |
|---|---|
| Samples | 7,180 |
| Integrated features | 27 |

## Tissue classification

Random Forest classification was applied to the integrated morphology matrix.

| Metric | Value |
|---|---|
| Accuracy | 0.95 |
| Balanced accuracy | 0.949 |
| Macro F1-score | 0.95 |

![Integrated confusion matrix](results/figures/random_forest_integrated_confusion_matrix.png)

Strong separation was observed for adipose tissue, background, debris and lymphocyte-rich regions. Residual overlap occurred mainly between tumour epithelium, normal mucosa and tumour-associated stroma, consistent with biologically plausible morphological similarity between related epithelial and stromal states.

## Tumour-stroma interface analysis

Tumour-stroma interface patches were identified using distances within morphology space. Patches with similar distances to tumour and stromal centroids were interpreted as morphologically ambiguous interface regions.

Interface score:

`abs(distance_to_tumour - distance_to_stroma)`

Lower values indicate stronger tumour-stroma ambiguity.

![Tumour-stroma interface montage](results/figures/tumour_stroma_interface_montage.png)

Interface-associated patches frequently show disrupted glandular organisation, spindle-like stromal components, extracellular matrix-rich morphology and features consistent with invasive or reactive tumour-associated regions.

## Tumour core versus interface morphology

Tumour-core and tumour-stroma-interface patches were compared quantitatively.

![Tumour interface feature comparison](results/figures/tumour_interface_feature_comparison.png)

Interface-associated tumour patches showed higher nuclei counts and smaller mean nuclear area compared with tumour-core patches, suggesting a more cellular and heterogeneous local microenvironment.

| Group | Nuclei count | Mean nucleus area | Nuclear area fraction | Entropy | Mean intensity |
|---|---:|---:|---:|---:|---:|
| Core tumour | 44.36 | 437.22 | 0.369 | 4.993 | 149.67 |
| Interface tumour | 75.72 | 220.33 | 0.300 | 4.887 | 151.51 |

## Normal-to-tumour morphology progression axis

A continuous normal-to-tumour morphology progression score was derived from the relative position of each patch to normal and tumour centroids in the integrated feature space.

| Score | Interpretation |
|---|---|
| Low score | Normal-like morphology |
| Intermediate score | Transitional morphology |
| High score | Tumour-like morphology |

![Normal-tumour progression UMAP](results/figures/normal_tumour_progression_umap.png)

Mean progression scores:

| Class | Mean score |
|---|---:|
| NORM | 0.401 |
| TUM | 0.564 |

The overlap between normal and tumour score distributions indicates that colorectal tissue morphology is not fully discrete at patch level, but forms a continuum between normal mucosal organisation and tumour-associated morphology.

## Normal-to-tumour progression atlas

Representative patches ordered along the inferred morphology progression axis.

![Normal-to-tumour progression atlas](results/figures/normal_to_tumour_progression_atlas.png)

Low progression scores correspond to organised glandular architecture, preserved epithelial polarity and clear luminal structures. Intermediate scores show architectural distortion, increased cellular density and partial gland disruption. High scores correspond to tumour-dominated morphology with irregular gland formation, enlarged nuclei and loss of normal tissue organisation.

## Key findings

1. Integrated morphology and nuclear features achieved a balanced classification accuracy of approximately 95% across nine colorectal tissue compartments.

2. Nuclear architecture contributed substantially to tissue discrimination beyond image-level morphology alone.

3. Tumour, stromal and normal epithelial compartments occupied partially overlapping regions of morphology space, while adipose tissue, lymphocyte-rich regions and debris formed highly distinct territories.

4. Tumour–stroma interface regions were identifiable as morphologically ambiguous patches located between tumour and stromal neighbourhoods in the integrated feature space.

5. Interface-associated tumour patches exhibited increased nuclei density and reduced mean nuclear area compared with tumour-core regions.

6. Progression-score distributions indicated continuous morphological variation between normal mucosa and tumour epithelium, with intermediate states occupying the space between these endpoints.

## Reproducibility

The full analysis is implemented as a Snakemake workflow.

Run the workflow:

`snakemake -s workflow/Snakefile --cores 4`

Dry run:

`snakemake -s workflow/Snakefile --cores 1 -n`
